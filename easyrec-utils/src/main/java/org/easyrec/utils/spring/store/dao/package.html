Provides utilities for DAO implementation. Some of these utilities help with common problems in JDBC code, like e.g. fetching an integer from a ResultSet with null checking, which normally would be coded like this: 
 Integer value = rs.getInt(columnName);
 if (rs.wasNull()) {
  	value = null;
 }
with {@link at.researchstudio.sat.util.dao.DAOUtils}, this becomes 
 Integer value =	DAOUtils.getInteger(rs, columnName);

Other parts of this package work together with the package at.researchstudio.sat.testutil.dao in project sat-testutil, the idea of which is that DAO classes can be tested in their normal environment by simply exchanging their configured DataSource. DAOs that are to be treated this way must be annotated with {@link at.researchstudio.sat.util.store.dao.DAO}.

Yet another part of this package is dedicated to standardizing DAO behaviour via aspect-oriented programming (see package {@link at.researchstudio.sat.util.store.dao.aop}). This mechanism is used to catch certain exceptions that are thrown in DAO code, and which originate in JDBC code of the spring framework, and return null instead. The idea is that all DAOs return null if no object is found and only throw exceptions if something goes wrong.
